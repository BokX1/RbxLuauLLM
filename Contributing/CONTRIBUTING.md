# Contributing

Thanks for contributing to RbxLuauLLM!

This project is intended for **authorized/dev/admin/testing contexts only**.

---

## Ground rules

- **No secrets in commits**: API keys, tokens, private endpoints, credential-like strings.
- **No misuse enablement**: donâ€™t add instructions or code aimed at bypassing permissions/ToS.
- Keep PRs small and focused.
- Update docs when behavior changes.

---

## How to contribute

1. Fork the repo
2. Create a branch: `feature/<name>` or `fix/<name>`
3. Make changes
4. Open a PR describing:
   - what changed
   - why it matters
   - screenshots/GIFs for UI changes

---

## What we welcome

- Prompt improvements and reduced hallucinations
- Validation and guardrails
- UI/UX improvements (including accessibility)
- Documentation and examples
- Adapters/bridges for authorized systems

---

## PR checklist

- [ ] No secrets/keys committed
- [ ] Handles errors gracefully
- [ ] Docs updated (if needed)
- [ ] Screenshots included (UI changes)
